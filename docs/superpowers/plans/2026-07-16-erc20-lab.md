# ERC-20 Lab Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and publish the complete Korean ERC-20 learning MVP described in the approved design.

**Architecture:** A plain Vite/TypeScript single-page application uses the Vanilla DOM API, a small observable store, and a History API router. Pure domain modules cover simulation, validation, persistence, and token generation; browser adapters cover solc Web Worker compilation and injected-wallet viem calls.

**Tech Stack:** Vite, TypeScript, Vanilla DOM API, Vanilla CSS, Solidity, viem, solc-js, CodeMirror 6, OpenZeppelin Contracts, Vitest.

## Global Constraints

- Do not use React, Vue, Svelte, Tailwind CSS, Bootstrap, a UI component library, or an external state library.
- Use `bigint` for token quantities and viem helpers for unit conversion and address validation.
- Never accept or store a private key or recovery phrase; enable writes only on Sepolia.
- Store only validated learning progress, drafts, edited code, deployment records, and recent hashes in `localStorage`.
- Keep remote Solidity imports disabled and compile only bundled OpenZeppelin sources.
- Use Korean beginner-friendly explanations beside technical errors.
- Work and push directly on `main` as requested.
