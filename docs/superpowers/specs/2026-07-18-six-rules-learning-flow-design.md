# THE SIX RULES 학습 흐름 재구성 설계

## 목표

개념 이해와 함수 실습의 경계를 명확히 나눈다. 개념 페이지는 ERC-20 개념 학습과 다음 단계 CTA로 끝내고, 여섯 가지 핵심 함수 선택과 decimals 설명은 함수 실습 페이지에서 제공한다.

## 정보 구조

### 상단 내비게이션

- `개념`, `함수 실습`, `코드 랩`, `토큰 만들기` 메뉴를 유지한다.
- `함수 실습`은 `/functions`로 연결한다.

### 개념 페이지 `/learn`

- 현재 개념 카드와 대체 가능성 실습은 유지한다.
- `THE SIX RULES` 라벨, 함수 카드 6개, decimals 설명 카드는 제거한다.
- 페이지 마지막에는 `함수 실습 시작` CTA를 배치한다.
- CTA를 누르면 STEP 1 완료 상태를 저장하고 `/functions`로 이동한다.

### 함수 실습 페이지 `/functions`

- 상단에 `STEP 2 · 함수 실습`을 작은 eyebrow 텍스트로 표시한다.
- `THE SIX RULES` 라벨과 `ERC-20의 여섯 가지 핵심 함수를 눌러보세요.` 제목을 표시한다.
- `totalSupply`, `balanceOf`, `transfer`, `approve`, `allowance`, `transferFrom` 카드 6개를 개념 페이지와 동일한 3열 카드 스타일로 표시한다.
- 함수 카드는 기존 `/functions/:key` 상세 실습으로 연결한다.
- 함수 카드 아래에는 `표시 수량`, `decimals`, `컨트랙트 내부 정수` 설명 카드를 배치한다.

### 함수 상세 페이지 `/functions/:key`

- 기존 뒤로가기, READ/WRITE 배지, 함수명, 시뮬레이션 UI를 유지한다.
- 뒤로가기 다음, READ/WRITE 배지 앞에 `STEP 2 · 함수 실습` eyebrow를 추가한다.
- 뒤로가기 fallback은 `/functions`를 유지한다.

## 컴포넌트와 데이터 흐름

- `renderLearnPage`는 함수 목록 데이터에 의존하지 않고 개념 콘텐츠와 CTA만 렌더링한다.
- `renderFunctionsPage`가 `FUNCTION_LESSONS`를 사용해 함수 카드 6개를 렌더링한다.
- 기존 `onComplete` 콜백이 STEP 1 완료 저장과 `/functions` 이동을 담당한다.
- 함수 상세 라우트와 시뮬레이터 동작은 변경하지 않는다.

## 반응형과 접근성

- 함수 카드 그리드는 기존 `.function-link-grid` 반응형 규칙을 재사용한다.
- CTA는 기존 버튼 컴포넌트와 키보드 동작을 유지한다.
- 카드 링크는 실제 `href`를 유지해 키보드와 브라우저 기본 동작을 지원한다.
- 함수 상세 STEP 2 표시는 시각적 맥락만 추가하며 제목 계층은 변경하지 않는다.

## 테스트

- 개념 페이지에 `THE SIX RULES`와 함수 카드가 없고 `함수 실습 시작` CTA가 있는지 확인한다.
- CTA 클릭 시 `/functions` 화면과 STEP 1 완료 상태가 확인되는지 테스트한다.
- 함수 실습 페이지에 STEP 2, THE SIX RULES, 함수 카드 6개, decimals 설명이 렌더링되는지 확인한다.
- 함수 상세 페이지에 `STEP 2 · 함수 실습`이 표시되는지 확인한다.
- 기존 함수 실행, 상태 변화, 코드 보기 테스트가 계속 통과해야 한다.

## 성공 기준

- 개념 페이지와 함수 실습 페이지 사이에 중복된 THE SIX RULES 콘텐츠가 없다.
- 사용자는 개념 페이지의 CTA 또는 상단 메뉴로 함수 실습 페이지에 진입할 수 있다.
- 함수 실습 페이지에서 여섯 함수 중 하나를 선택해 기존 상세 실습으로 이동할 수 있다.
- 모든 관련 페이지에서 STEP 1과 STEP 2 맥락이 일관되게 보인다.
- 전체 타입 검사, 테스트, 프로덕션 빌드가 통과한다.
