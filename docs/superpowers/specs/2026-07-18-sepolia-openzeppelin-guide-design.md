# Sepolia·OpenZeppelin 안내 페이지 설계

## 목표

공통 푸터의 안전 안내 문구를 교육 링크로 교체하고, Web3 초보자가 토큰 배포에 사용되는 Sepolia와 OpenZeppelin의 역할 및 관계를 한 페이지에서 이해하도록 한다.

## 범위

- 공통 푸터에 `Sepolia와 OpenZeppelin 알아보기 →` 내부 링크를 표시한다.
- `/guide/sepolia-openzeppelin` 경로에 독립 교육 페이지를 추가한다.
- 새 페이지에서 Sepolia와 OpenZeppelin을 각각 설명하고, 두 기술이 토큰 배포 과정에서 어떻게 연결되는지 보여준다.
- Ethereum과 OpenZeppelin의 공식 문서 링크를 제공한다.
- 새 페이지 상단에 기존 함수 상세 페이지와 같은 형태의 `<` 이전 버튼을 제공한다.
- 이번 변경은 새로운 지갑 기능, 네트워크 요청, 사용자 상태 저장을 추가하지 않는다.

## 콘텐츠 구조

1. **페이지 소개**
   - 눈썹 제목: `DEPLOYMENT BASICS`
   - 제목: `배포 전에 알아둘 두 가지`
   - 보조 문구: 토큰 코드와 테스트 네트워크가 서로 다른 역할을 한다는 점을 안내한다.
2. **Sepolia**
   - Ethereum 애플리케이션과 스마트 컨트랙트를 시험하는 공개 테스트넷이다.
   - 테스트넷 ETH는 실습용이며 실제 금전적 가치를 전제로 하지 않는다.
   - 메인넷과 계정 형식 및 실행 방식은 비슷하지만 잔액과 거래 기록은 분리된다.
3. **OpenZeppelin**
   - ERC-20 같은 표준 구현을 제공하는 재사용 가능한 Solidity 컨트랙트 라이브러리다.
   - 검증된 기반 코드를 활용해 직접 구현에서 생길 수 있는 실수를 줄인다.
   - 라이브러리 사용이 프로젝트별 검토와 테스트를 대체하지는 않는다.
4. **함께 사용하는 흐름**
   - `OpenZeppelin으로 ERC-20 코드 구성 → 컴파일 → Sepolia에 배포 → 지갑과 탐색기에서 확인`
5. **공식 문서**
   - Ethereum 네트워크 문서: `https://ethereum.org/developers/docs/networks/`
   - OpenZeppelin Contracts 5.x 문서: `https://docs.openzeppelin.com/contracts/5.x`

## 화면과 상호작용

- 기존 Coinbase 기반 디자인 토큰, 타이포그래피, 카드, 여백을 재사용한다.
- 데스크톱에서는 Sepolia와 OpenZeppelin 설명을 2열 카드로, 좁은 화면에서는 1열로 표시한다.
- 두 기술의 관계는 별도의 흐름 카드로 표시하되 이미지나 새 아이콘 의존성은 추가하지 않는다.
- 푸터 링크는 SPA 내부 링크로 동작한다.
- 공식 문서 링크는 새 탭에서 열고 `rel="noreferrer noopener"`를 적용한다.
- 이전 버튼은 브라우저 방문 기록이 있으면 `history.back()`을 사용한다. 직접 URL로 진입한 경우 홈(`/`) 링크가 기본 대체 경로가 된다.

## 코드 구성

- `src/pages/deploymentBasicsPage.ts`: 교육 페이지 마크업과 외부 문서 링크를 담당한다.
- `src/app.ts`: 새 경로를 연결하고 공통 푸터 문구를 내부 링크로 교체한다.
- 기존 `routeLink`와 `function-back-link` 스타일을 재사용해 라우팅 및 이전 버튼의 동작과 외형을 일관되게 유지한다.
- 페이지 전용 배치가 필요한 경우 기존 페이지 스타일시트에 최소한의 클래스만 추가한다.

## 데이터 흐름과 오류 처리

- 정적 교육 콘텐츠만 렌더링하므로 네트워크 로딩과 런타임 오류 상태는 없다.
- 알 수 없는 경로는 현재 앱 동작대로 홈 화면으로 처리한다.
- 공식 문서 사이트가 일시적으로 열리지 않아도 앱 내부 교육 콘텐츠는 그대로 읽을 수 있다.

## 테스트

- 공통 푸터에 새 내부 링크가 있고 기존 안전 문구가 제거되는지 확인한다.
- 새 경로에서 Sepolia, OpenZeppelin, 함께 사용하는 흐름, 공식 문서 링크가 렌더링되는지 확인한다.
- 이전 버튼의 접근 가능한 이름과 홈 대체 경로를 확인한다.
- 방문 기록이 있는 경우 이전 버튼이 `history.back()`을 호출하는지 확인한다.
- 전체 타입 검사, 테스트, 프로덕션 빌드를 실행한다.

## 완료 기준

- 모든 화면의 공통 푸터에서 새 교육 페이지로 이동할 수 있다.
- 초보자가 한 페이지에서 두 기술의 차이와 연결 관계를 이해할 수 있다.
- 키보드와 스크린리더 사용자가 푸터 링크, 이전 버튼, 공식 문서 링크를 식별할 수 있다.
- 기존 학습·함수 실습·코드 랩·토큰 만들기 흐름에 회귀가 없다.
