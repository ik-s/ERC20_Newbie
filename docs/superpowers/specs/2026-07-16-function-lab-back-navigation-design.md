# 함수 실습 상세 화면 뒤로가기 설계

## 목표

모든 ERC-20 함수 실습 상세 화면의 히어로 상단에 이전 화면으로 돌아가는 `<` 컨트롤을 제공한다. 사용자가 함수 목록에서 상세 화면으로 이동한 일반적인 흐름과 상세 URL로 직접 들어온 흐름을 모두 지원한다.

## 사용자 경험

- 위치: `.page-hero.function-hero`의 첫 번째 요소로 배치한다.
- 형태: 현재 Coinbase 기반 디자인과 어울리는 둥근 아이콘 링크로 표시한다.
- 라벨: 화면에는 `<`를 표시하고 접근성 이름은 `이전 화면으로 돌아가기`로 제공한다.
- 입력 방식: 마우스, 터치, 키보드로 사용할 수 있어야 한다.
- 반응형: 데스크톱과 모바일에서 최소 44px의 클릭 영역을 유지한다.

## 이동 동작

컨트롤은 실제 링크이며 기본 목적지는 `/functions`다.

1. 현재 탭에 이전 방문 기록이 있으면 기본 링크 이동을 막고 `window.history.back()`을 실행한다.
2. 이전 방문 기록이 없으면 기존 SPA 라우터가 링크의 `/functions` 목적지를 처리한다.
3. 따라서 함수 목록에서 들어온 사용자는 원래 화면으로 돌아가고, 상세 URL로 직접 들어온 사용자는 함수 목록으로 이동한다.

## 구현 범위

- 공통 함수 상세 렌더러인 `renderFunctionLabPage`에 컨트롤을 한 번 추가해 모든 함수 상세 화면에 적용한다.
- 기존 `routeLink`를 재사용해 fallback URL과 SPA 내비게이션을 유지한다.
- 전용 CSS 클래스로 크기, 테두리, hover, focus-visible 상태를 정의한다.
- 라우터 구조와 다른 페이지의 내비게이션은 변경하지 않는다.

## 접근성

- `<a href="/functions">` 시맨틱을 유지한다.
- `aria-label="이전 화면으로 돌아가기"`를 지정한다.
- `:focus-visible` 상태에서 명확한 포커스 링을 표시한다.
- 색상만으로 상태를 전달하지 않는다.

## 테스트

- 함수 상세 화면에 뒤로가기 링크가 한 개 렌더링되는지 확인한다.
- 링크의 `href`가 `/functions`이고 접근성 이름이 올바른지 확인한다.
- 직접 진입 조건에서 링크 클릭 시 함수 목록 화면으로 이동하는지 확인한다.
- 기존 함수 실행, 상태 변화, 코드 보기 테스트가 계속 통과해야 한다.

## 성공 기준

- 모든 `/functions/:key` 화면에서 히어로 상단에 `<` 컨트롤이 보인다.
- 이전 기록이 있는 일반 흐름에서는 직전 화면으로 돌아간다.
- 이전 기록이 없는 직접 진입에서는 `/functions`로 이동한다.
- 데스크톱과 모바일에서 컨트롤이 겹치거나 잘리지 않는다.
- 전체 타입 검사, 테스트, 프로덕션 빌드가 통과한다.
